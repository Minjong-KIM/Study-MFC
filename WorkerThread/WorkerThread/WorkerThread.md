# 멀티 스레드
응용프로그램이 여러 개의 스레드를 동시에 실행하는 것. 스레드는 CPU 시간을 할당받아 실행되는 동적인 개념이다. 여러 개의 스레드가 한 개의 CPU에서 동시에 실행되려면 실행중인 스레드가 중지(SUSPEND)하고 상태를 저장하는 작업과 새로 실행할 스레드의 상태를 복원하는 작업을 빨리 해줘야한다.
# CPU 스케줄링
1. 우선순위 클래스 -> 같은 프로세스가 생성한 스레드는 모두 동일한 우선순위를 가진다.
2. 우선순위 레벨 -> 같은 프로세스에 속한 스레드 간 상대적 우선순위를 결정한다.
# 스레드 동기화
멀티 스레드를 사용하는 프로그램에서 두 개 이상의 스레드가 공유 데이터에 접근하면 문제가 발생할 수 있는데 이런 문제를 해결하는 것을 스레드 동기화라 부른다.
# 작업자 스레드
메시지 루프가 없어 화면에 보이지 않는 백그라운드 작업에 적합하다.
## CWinTread* AfxBeginThread(제어함수, 제어 함수 실행시 전달할 값, 스레드 우선순위,스택 사이즈, 스레드 생성 제어 옵션, 보안설명자)
1. 제어 함수 -> 스레드를 실행하는 시작점이 되는 함수를 가리키는 포인터로" UINT 함수명" 형태여야 한다.
2. 제어 함수 실행시 전달할 값 ->제어 함수는 포인터 크기의 값 하나만 받을 수 있다.
3. 스레드 우선순위 -> ex) THREAD_PRIORITY_NORMAL
4. 스레드 생성 제어 옵션 ->ex) CREATE_SUSPENDED 는 스레드 생성후 바로 실행되지 않는다.
