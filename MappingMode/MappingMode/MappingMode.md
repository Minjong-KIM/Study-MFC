# MappingMode
1. 대부분의 GDI 함수에서 사용하는 단위, 즉 프로그래머가 직접 사용하는 단위는 논리 단위이다.
2. 운영체제 내부적으로 사용하는 단위는 픽셀을 기본으로 하는 장치 단위이다.
3. 프로그래머가 사용하는 논리 단위를 장치 단위로 변환하는 방법을 Mapping Mode라고 한다.
4. MM_TEXT는 DC를 최초에 생성했을 때 자동으로 설정되어 있는 매핑모드로 논리 단위와 장치 단위가 1대 1로 대응한다.
5. MM_TEXT를 사용하여 그림을 그리면 장치에 따라 그림의 크기가 달라진다.
6. MM_LOMETRIC은 논리 단위와 장치 단위가 1대 0.1mm로 대응한다. 또한 클라이언트 영역이 테카르트 좌표계의 4분면에 해당한다.
7. MM_HIMETRIC은 MM_LOMETRIC과 같지만 논리 단위와 장치 단위가 1대 0.01mm로 대응한다.
8. MM_ANISOTROPIC은 두 함수를 이용해 논리 단위와 장치 단위의 변환 관계를 직접 정의한다. CDC::SetWindowExt() 함수로 논리 단위의 범위를 지정하고CDC::SetViewPortExt() 함수로 장치 단위의 범위를 지정한다.
