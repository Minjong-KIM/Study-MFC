# 스레도 동기화
1. CCriticalSection -> 공유 자원에 접근하는 다수의 스레드가 있을 때 오진 한 개의 스레드만 접근할 수 있게 해준다. "CCriticalSection 객체.Lock()" 함수를 먼저 호출하는 쪽이 공유 자원에 접근할 수 있고 Unlock() 함수를 호출해주면 다른 스레드가 공유 자원에 접근할 수 있다. 서로 다른 프로세스에 속한 스레드간 동기화에는 사용할 수 없다.
2. CMutex -> CCriticalSection과 동일하지만, 커널모드로 동작하므로 속도는 상대적으로 느리지만 서로 다른 프로세스에 속한 스레드간 동기화에 사용할 수 있다.
3. CEvent -> 신호와 비신호 두 개의 상태를 가진 동기화 객체이다.
4. CSemaphore -> 자원에 접근할 수 있는 스레드 수를 제어하는 동기화 객체이다.
