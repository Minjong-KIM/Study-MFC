# Menu
1. 최상위 메뉴(메뉴바) -> 일반적으로 메뉴라고 할 때 가리키는 메뉴
2. 드랍다운 메뉴 -> 최상위 메뉴를 선택하면 펼쳐지는 메뉴
3. 컨텍스트 메뉴 -> 마우스 오른쪽 버튼을 클릭하면 열리는 메뉴
4. 팝업 메뉴 -> 사용자 행위의 결과로 나오는 메뉴
5. 시스템 메뉴 -> 타이틀바 왼쪽 아이콘을 클릭하면 나오는 메뉴
6. 메뉴 항목 -> 명령 항목과 팝업 항목으로 명령 항목은 WM_COMMAND 메시지가 발생하고 팝업 항목은 발생하지 않는다.
7. 엑세스키 -> 메뉴가 열린 상태에서 특정 항목을 키보드로 빠르게 선택할 수 있게 해준다.
8. 단축키 -> 메뉴가 열리지 않은 상태에서 키 조합으로 메뉴 항목의 기능을 바로 실행할 수 있게 한다.
# 메뉴 생성
메뉴 생성은 리소스를 정의 하여 생성할 수 있고 코드를 이용해 생성할 수 있다. 주로 리소스 정의 방법을 사용하며 코드를 이용한 방법은 메뉴를 추가하거나 수정할 때 사용한다.
# 리소스를 이용해 메뉴 만들기
1. Prompt -> '\n'을 기준으로 앞쪽 문자열은 상태바에 표시되고, 뒤쪽 문자열은 툴팁으로 표시된다.
2. Separator(구분 기호) -> 메뉴 항목을 구분하는 가로줄을 표시한다.
3. Break(나누기) -> 메뉴 항목이 많을 때 column 또는 bar를 선택해 나눠준다.
4. Caption -> 액세스 키는 해당 문자 앞에 '&'기호를 붙여주고, 단축키는 '\t'기호를 삽입해 단축키를 작성한다. ex) ***(&T)\tCtrl+X
5. Checked(확인) -> 메뉴 항목에 체크 표시를 한다.
6. Enabled(사용) -> 이 값이 False면 표시는 되지만 사용하지 못한다.
7. Grayed(회색 표시) -> 항목이 흐리게 되어 사용하지 못한다.
8. Pop-Up -> 명령 항목이 아닌 팝업 항목이 된다. 최상위 메뉴가 대게 Pop-Up메뉴다.
# 코드를 이용해 메뉴 만들기
1. CMenu 객체를 만든다.
2. CreateMenu() 함수를 이용해 메뉴 객체와 연결한다.
3. AppendMenu() 함수로 메뉴 항목을 추가한다.
4. Detach() 함수를 호출해 메뉴 객체와 메뉴를 분리한다. 이유는 윈도우 객체가 파괴되어도 윈도우가 파괴되지 않게 하기 위해서이다.
# 메뉴 명령 처리하기
팝업 항목이 아닌 명령 항목의 경우 WM_COMMAND 메시지를 발생 시키기 때문에 WM_COMMAND 메시지 핸들러를 작성해야 한다.
